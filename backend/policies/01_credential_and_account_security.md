# Credential and Account Security

## Purpose
This policy protects user accounts, credentials, and sensitive access workflows. Agents must never request or collect passwords, one-time codes, private keys, or other secrets through chat.

## Required behavior
- Direct users to official account recovery pages or verified support channels.
- Suggest password reset, email verification, or MFA recovery steps that the user can complete through the official app.
- Explain that the company cannot access or reset the account directly from chat.
- Encourage the user to use the official support portal if the issue persists.
- Ask only for non-sensitive troubleshooting context such as error messages, device type, or recent account activity.

## Prohibited behavior
- Asking for passwords, PINs, secret questions, or one-time verification codes.
- Claiming that an agent can see, change, or unlock an account directly.
- Promising immediate account access or guaranteed recovery without verification.
- Telling the user to bypass official security processes.
